Reference: IVELoadPlugin

Overview

Dynamically loads a plugin.

Prototype (C)

#include "ive_plugin.h"
IVEPlugin IVELoadPlugin(const char* path, const char* init_symbol_name, const char* interface_name);

Prototype (Fortran)

There is no Fortran interface.

Parameters

path
(in) path is the path name of the dynamic object with the code to be loaded. If path is not a full file name, the search algorithm used, if any, depends on the underlying operating system.
init_symbol_name
(in) init_symbol_name is the symbol name for the function IVELoadPlugin will call to initialize the plugin. Note that the symbol name is the name stored in the object file without any system-specific adornment (namely the leading underscore prepended on Mac OS X systems). If your initialization function is in C++, you will likely want to complile it wtih C linkage to avoid having to pass a mangled name for init_symbol_name. If init_symbol_name is NULL, IVELoadPlugin assumes that the symbol name is initialize_plugin. For the expected prototype and behavior of the initialization function, consult the IVEPluginInitProc description.
interface_name
(in) If the plugin supports multiple interfaces, pass the name of the desired interface as interface_name; otherwise pass NULL for interface_name.

Return value

If IVELoadPlugin successfully loads the plugin, a non-NULL handle to the plugin is returned. To access the plugin's functions or data, use that handle as an argument to IVEGetPluginAttr to get the address of the plugin's attributes. To unload the plugin and free resources allocated when the plugin was loaded, use the handle as the argument to IVEUnloadPlugin. If IVELoadPlugin fails, it returns NULL and you may use IVEGetPluginErrorMessage to get a description of the reason for the failure.

Cross references
