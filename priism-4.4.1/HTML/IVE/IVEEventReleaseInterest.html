Reference: IVEEventReleaseInterest

Overview

With a multi-threaded event context, the invocation of a callback routine for an event on a file descriptor temporarily prevents the same type of event on the same file descriptor from triggering any more callbacks. A fuller description of that temporary block can be found in the documentation for IVEInputProc, but, typically, events will once again trigger callbacks when the callback routine returns, or the thread executing that callback routine calls one of the event handling functions, IVEEventLoop or IVEProcessEvent. To reenable the link between events and callbacks without returning from the callback or invoking one of the event handling functions, the thread executing the callback should call IVEEventReleaseInterest. Calling IVEEventReleaseInterest when not in the process of executing a callback routine or with a single-threaded event context has no effect.
