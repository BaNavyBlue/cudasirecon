Reference: IVEEventContext

Overview

In C, an IVEEventContext is an opaque pointer to the structure which stores information about event sources and their associated callbacks. Include ive_eventloop.h to declare IVEEventContext.

Event contexts and multiple threads

Reliable use of an event context from multiple threads of execution is only possible if the context was created with IVECreateThreadedEventContext. The following caveats apply when using a context with multiple threads:

  • You'll have to synchronize your threads appropriately so a call to IVEDestroyEventContext does not occur while other threads are executing event callbacks which will return. The documentation for IVEDestroyEventContext describes one way to handle the synchronization.
  • IVEDestroyEventContext can not be safely called on a multi-threaded event context from a destructor registered by pthread_key_create() because the event context makes use of its own thread local storage with a destructor and the order in which multiple destructors are called for a thread is not specified.
  • In a pthread environment, deferred cancellation of a thread invoking one of the event processing functions will leave the context in a usable state. Asynchronous cancellation of a thread invoking one of the event processing functions is not guaranteed to leave the context in a usable state.
