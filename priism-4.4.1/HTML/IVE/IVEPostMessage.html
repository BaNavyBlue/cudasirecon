Reference: IVEPostMessage

Overview

Generates an event to trigger a callback registered with IVESetMessageHandler. Whether or not a callback is registered is checked when the message is processed (i.e. in a call to IVEProcessEvent or IVEEventLoop.html) and not when the message is posted. If no callback is registered for the message type when the message is processed, the event processing simply discards the message (and will not release a thread from a blocking call to IVEProcessEvent).

Prototype (C)

#include "ive_eventloop.h"
int IVEPostMessage(IVEMessage msg, IVEEventContext context);

Prototype (Fortran)

integer function IVEPostMessage(msg, context)
integer msg, context

Parameters

msg
desc is the integer identifying the type of message being posted. msg should be between IVE_MESSAGE_MIN and IVE_MESSAGE_MAX; if it is not, the call to IVESetMessageHandler has no effect. IVE_MESSAGE_MIN and IVE_MESSAGE_MAX are defined in ive_eventloop.h for C and ive_eventloop.inc for Fortran.
context
context is the event context to modify. event source will be added.

Return value

Returns zero if the message was posted successfully. Returns one if the message could not be posted successfully and a future call to IVEPostMessage with the same values for msg or context will also fail. That could happen for one of two reasons: msg is less than IVE_MESSAGE_MIN or greater than IVE_MESSAGE_MAX or the state of the event context has been corrupted. Otherwise returns two if the message could not be posted successfully but a future call may succeed if event processing has drained the backlog of events in the event queue.
