Reference: IVESetMessageHandler

Overview

Installs a callback that will be invoked for events generated by IVEPostMessage. The intent of IVESetMessageHandler and IVEPostMessage is to allow events to be triggered by internal state changes in addition to the external triggers (I/O, signals, or timers) already available. The newly installed callback replaces whatever callback had been installed previously

Prototype (C)

#include "ive_eventloop.h"
void IVESetMessageHandler(IVEMessage msg, IVEMessageProc proc, void* arg, IVEEventContext context);

Prototype (Fortran)

subroutine IVESetMessageHandler(msg, proc, arg, context)
integer msg, context
integer arg(*)
external proc

Parameters

msg
desc is the integer identifying which messages will be handled by the procedure proc. msg should be between IVE_MESSAGE_MIN and IVE_MESSAGE_MAX; if it is not, the call to IVESetMessageHandler has no effect. IVE_MESSAGE_MIN and IVE_MESSAGE_MAX are defined in ive_eventloop.h for C and ive_eventloop.inc for Fortran.
proc
A thread which calls IVEProcessEvent or is waiting in an event loop can invoke proc after IVEPostMessage has been used to post a message of type msg. The documentation for IVEMessageProc describes the arguments passed to proc.
arg
arg is not used internally but is passed along as an argument to proc.
context
context is the event context to modify.
