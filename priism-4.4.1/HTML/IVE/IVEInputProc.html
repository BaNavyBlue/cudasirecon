Reference: IVEInputProc

Overview

In C, IVEInputProc is the prototype for callback routines registered with IVEAddInput. As shown below, it takes four arguments: the file descriptor (handle) that is ready, the argument registered when the callback was registered, the handle assigned to the callback, and the event context. The Fortran interface is similar and is also shown below.

Include ive_eventloop.h to declare IVEInputProc.

Prototype (C)

void (*IVEInputProc)(IVEDesc desc, void* arg, IVEInputId id, IVEEventContext context);

Prototype (Fortran)

subroutine your_name_here(desc, arg, id, context)

integer arg(*)

integer desc, id, context

Parameters

desc
desc is the file descriptor (handle) for which id was registered.
arg
arg is the argument that was registered with IVEAddInput when the callback routine was registered.
id
id is the ID assigned to the callback when it was registered.
context
context is the event context in which the event occurred.

Side effects with multiple threads

With a context used by multiple threads, the occurrence of an event (ready for reading, ready for writing, or available out-of-band data) on a file descriptor will cause the registered callback routine to be invoked, but no more callbacks will be generated for an event of the same type on the same file descriptor until one of the following occurs:

    the previously invoked callback routine returns

    the thread executing the previously invoked callback routine calls IVEEventLoop, IVEEventReleaseInterest, or IVEProcessEvent

    the thread executing the previously invoked callback routine exits

    a call, by any thread, to IVEAddInput registers a callback for events on that file descriptor with reasons that include the cause for the previously invoked callback
