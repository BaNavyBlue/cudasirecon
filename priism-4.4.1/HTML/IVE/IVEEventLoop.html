Reference: IVEEventLoop

Overview

For a context used by a single thread, IVEEventLoop will enter an infinite loop to process incoming events. In that situation, it has the same effect as the following snippet of C: 

    while (1) {
        (void) IVEProcessEvent(1, context);
    }

For a context used by multiple threads, IVEEventLoop will enter a loop to process events and will only return if the event context has been destroyed with IVEDestroyEventContext.

Prototype (C)

#include "ive_eventloop.h"
void IVEEventLoop(IVEEventContext context);

Prototype (Fortran)

subroutine IVEEventLoop(context)
integer context

Parameters

context
context is the context from which to draw events.

Side effects

When the event loop processes a file descriptor event for a multi-threaded event context, the same type of events on the same file descriptor are temporarily blocked from generating any more calls to callback routines. The documentation for IVEInputProc describes the conditions that end the blockade.

Cross references
