Reference: IVEArenaAttr

Overview

In C, an IVEArena represents a shared memory segment. IVECreateArena creates an arena and IVEAttachArena attaches to one that already exists. The IVE library provides a set of calls for dynamic allocation of memory from the shared memory (IVEShmAlloc and IVEShmCalloc). There is also support for mutual-exclusion and read-write locks to synchronize access.

Because an IVEArena is not guaranteed to be mapped to the same address in all the processes that use the arena, pointer values should not be directly exchanged between processes. Use IVEPtrToRep and IVERepToPtr to convert to and from a format that can be exchanged. Because it is not generally possible to know where functions are stored in the different processes, do not exchange function pointers or C++ structures which are not plain old data (PODs).

When unrelated processes share an IVEArena, they may not have a starting point (a shared IWCPtrRep to a data structure) with which to start communication. The calls IVEGetArenaRoot, IVESetArenaRoot, and IVECompareSwapArenaRoot can be used to provide this starting point.

Include ive_shm.h to declare IVEArena.
