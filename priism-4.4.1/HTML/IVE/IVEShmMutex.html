Reference: IVEShmMutex

Overview

An IVEShmMutex is used to synchronize access to a resource shared by multiple threads which may be in different processes. Only one thread can lock (via IVEShmAcquireMutex or IVEShmTryAcquireMutex) an IVEShmMutex at a time; other threads will wait. A thread which has locked an IVEShmMutex unlocks it with IVEShmReleaseMutex at which point the same or another thread can lock it. IVEShmMutex locks are not recursive; a thread which holds a lock on a IVEShmMutex can not it lock it again.

An IVEShmMutex can not be used interchangeably with an IVEThreadMutex; the latter only guarantees synchronization between threads in the same process.

Include ive_standards.h and ive_shm.h to declare IVEShmMutex. Beware that inclusion of ive_standards.h sets feature-set macros. It is safest to include it before any other header.
